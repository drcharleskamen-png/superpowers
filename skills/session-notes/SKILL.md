End this session by documenting what happened.

## 1. Session Summary
Write a brief summary of:
- What was accomplished
- Key decisions made and why
- Any problems encountered and how they were resolved
- What's left to do

## 2. Update Memory
Update the project's memory files:
- Add any new patterns, conventions, or lessons learned to `MEMORY.md`
- If a `CLAUDE.md` exists in the project root, update its "Rules & Lessons Learned" section with anything new

## 3. Update TODO
If there are unfinished tasks, list them clearly:
- What's blocked and why
- What should be done next session
- Priority order

## 4. Output
Present the session notes in a clean format the user can review.
Keep it concise — no more than 20 lines.
